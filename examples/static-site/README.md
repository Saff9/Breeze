# Static Site Generator in Breeze

A tiny static site generator written in [Breeze](../..). It reads a list of
posts (each one a Breeze object — a list of `[key, value]` pairs), renders
each post to HTML using helpers from `templates.bz`, wraps everything in a
full HTML5 document via `html.page()`, and writes the result to
`index.html`.

This is the same model Jekyll, Hugo, or Eleventy use — just in ~25 lines of
Breeze.

## Files

| File             | Purpose                                                        |
| ---------------- | ------------------------------------------------------------- |
| `main.bz`        | The generator: defines the posts, renders them, writes the page. |
| `templates.bz`   | `render_post(post)` and `render_list(items)` helpers.         |
| `index.html`     | Generated by running `main.bz` (not checked in).              |

## Run it

```bash
breeze run main.bz
```

You'll see:

```
Generated index.html (NNNN bytes)
Open index.html in your browser to see the result.
```

Then open `index.html` in your browser to see a small blog with three posts.

## What it demonstrates

- The `html` stdlib module (`html.h1`, `html.p`, `html.div`, `html.h2`,
  `html.page`).
- The `fs` stdlib module (`fs.write`).
- The `json` stdlib module (`json.get` for reading keys from a Breeze
  object — a list of `[key, value]` pairs).
- Multi-file programs: `main.bz` imports `render_post` and `render_list`
  from `templates.bz`.
- String interpolation: `"A blog generated by Breeze. {len(posts)} posts so far."`.
- The friendly `+` operator (auto string concatenation).

## Tinker

- Add a fourth post to the `posts` list and re-run.
- Replace `html.p("<em>" + date + "</em>")` with a real `<time>` tag using
  `html.tag("time", date, [["datetime", date]])`.
- Use `render_list` somewhere — for example, list all post titles at the
  top of the page.
